1. Por qué importan los Frameworks: Una Perspectiva Histórica

Muchos desarrolladores saltan directamente a React sin comprender los principios fundamentales de la web. Es como un pintor que intenta hacer arte abstracto sin haber aprendido nunca a dibujar de forma realista.

Consecuencia: Esto crea una mentalidad de "copiar y pegar" sin entender el porqué, lo que lleva a:

  • Incapacidad para depurar problemas complejos
  • Bajo rendimiento en entrevistas técnicas
  • Dificultad para distinguir a un senior de un junior

Por qué los Frameworks Revolucionaron el Desarrollo Web

La Crisis Pre-React

Antes de React y frameworks modernos, los desarrolladores enfrentaban un dilema clásico:

  • jQuery/DOM Vanilla: Total control pero caótico a escala
  • MVC Heavy (Backbone, Angular 1): Estructura pero mucho boilerplate
  • El Problema: Mantener la UI sincronizada con el estado era un infierno

React introdujo un cambio fundamental en el pensamiento:

// UI = f(estado)
// Esta simple ecuación cambió todo

const App = ({ users, isLoading }) => {
  if (isLoading) return <div>Cargando...</div>;
  return (
    <ul>
      {users.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
};

Antes de aprender React, aprende JavaScript puro. Entiende cómo funcionan los eventos, el DOM API, promesas, async/await. Sin esto, React se vuelve "magia" y no entenderás qué está sucediendo cuando algo falla.
