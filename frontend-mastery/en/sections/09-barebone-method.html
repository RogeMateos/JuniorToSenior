2.8 Barebone Method

The Barebone Method is a technique for building components first without styling, without optimizations, just pure functionality. Then you iterate and improve. It's especially useful for interviews and prototypes.

What is Barebone?

Barebone means "pure functionality". It's building a component with:

  • ‚úÖ Correct logic
  • ‚úÖ Correct state
  • ‚úÖ Basic error handling
  • ‚ùå No elaborate CSS
  • ‚ùå No performance optimizations
  • ‚ùå No complex edge cases

Example: Barebone vs Production Implementation

Step 1: Barebone (Pure Functionality)

// ‚úÖ Barebone: Functional but unpolished
const UserTable = ({ users, onSelect }) => {
  const [sortBy, setSortBy] = useState('name');

  const sortedUsers = [...users].sort((a, b) => {
    if (sortBy === 'name') return a.name.localeCompare(b.name);
    if (sortBy === 'age') return a.age - b.age;
    return 0;
  });

  return (
    <div>
      <div>
        <button onClick={() => setSortBy('name')}>Sort by Name</button>
        <button onClick={() => setSortBy('age')}>Sort by Age</button>
      </div>
      <table>
        <tr><th>Name</th><th>Age</th></tr>
        {sortedUsers.map(u => (
          <tr key={u.id} onClick={() => onSelect(u)}>
            <td>{u.name}</td>
            <td>{u.age}</td>
          </tr>
        ))}
      </table>
    </div>
  );
};

Step 2: Add Presentation

// ‚úÖ With styling
const UserTable = ({ users, onSelect }) => {
  const [sortBy, setSortBy] = useState('name');
  const [sortDirection, setSortDirection] = useState('asc');

  const sortedUsers = [...users].sort((a, b) => {
    let comparison = sortBy === 'name'
      ? a.name.localeCompare(b.name)
      : a.age - b.age;
    return sortDirection === 'asc' ? comparison : -comparison;
  });

  return (
    <div className="user-table-container">
      <div className="sort-controls">
        <button
          className={sortBy === 'name' ? 'active' : ''}
          onClick={() => {
            if (sortBy === 'name') {
              setSortDirection(d => d === 'asc' ? 'desc' : 'asc');
            } else {
              setSortBy('name');
              setSortDirection('asc');
            }
          }}
        >
          Sort by Name {sortBy === 'name' && (sortDirection === 'asc' ? '‚Üë' : '‚Üì')}
        </button>
      </div>
      <table className="user-table">
        <thead>
          <tr><th>Name</th><th>Age</th><th>Action</th></tr>
        </thead>
        <tbody>
          {sortedUsers.map(u => (
            <tr key={u.id} className="user-row">
              <td>{u.name}</td>
              <td>{u.age}</td>
              <td>
                <button
                  className="btn-select"
                  onClick={() => onSelect(u)}
                >
                  Select
                </button>
              </td>
            </tr>
          ))}
        </tbody>
      </table>
    </div>
  );
};

Barebone is building the foundation, walls, and roof (the essentials). Then you paint, install electricity, add furniture (presentation and polish). Don't try to decorate before the walls are up.

Advantages of the Barebone Method

  • Clarity: You focus on logic, not visual details
  • Testing: Easier to test pure functionality
  • Quick Iteration: Change logic easily without worrying about CSS
  • Interviews: Show functional code quickly without wasting time on styling
  • Collaboration: Backend developers can easily understand the logic

Use Barebone in coding interviews. Interviewers value seeing functional code quickly. You can say: "Here's the barebone functionality. In a real project, I'd add [list of improvements]".
