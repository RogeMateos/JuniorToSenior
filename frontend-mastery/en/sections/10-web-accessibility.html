2.9 Web Accessibility (A11Y)

Accessibility is not an "extra", it's a requirement. A senior developer builds accessible interfaces by default. A11Y (accessibility) is crucial for users with disabilities and for SEO.

WCAG 2.1 Principles

1. Perceivable

Information must be presented in ways everyone can perceive:

  • Sufficient contrast between text and background (4.5:1 for normal text)
  • Image descriptions (alt text)
  • Transcripts for videos

2. Operable

The interface must be navigable with keyboard:

  • Everything accessible via Tab
  • Buttons respond to Enter/Space
  • Links should be distinguishable
  • Skip links to bypass repetitive navigation

3. Understandable

Content must be clear and predictable:

  • Simple language
  • Clear form instructions
  • Predictability: if I do something, I expect X to happen

4. Robust

Compatible with assistive technologies:

  • Semantic HTML
  • ARIA attributes when necessary
  • Screen reader compatibility

Common Mistakes

  • Not using labels in forms
  • Using divs for buttons instead of <button>
  • No alt attributes on images
  • Color as the only way to communicate
  • Not testing with screen readers (NVDA, JAWS)
  • Zoom smaller than 200% causing overflow
  • No keyboard navigation support

A wheelchair ramp benefits not just people in wheelchairs. It also helps parents with strollers, elderly people, and people with luggage. Accessibility benefits EVERYONE.

A11Y is not optional. It's legally required in many jurisdictions (under ADA in the US, AODA in Canada). It's a sign of professionalism. Senior developers build accessible interfaces by default.
