5Ô∏è‚É£ First Day on a Linux Server (Minimum You Should Know)

When you create an instance:

  • Connect via SSH
  • Update packages
  • Create non-root user
  • Configure firewall
  • Install dependencies

üîê Basic Security (Mandatory)

Minimum checklist:

  • SSH with key (not password)
  • Disable root login
  • Firewall active
  • Open only 22, 80, 443

Conceptual example:

sudo adduser deploy
sudo usermod -aG sudo deploy
sudo ufw allow OpenSSH
sudo ufw allow 80
sudo ufw allow 443
sudo ufw enable

üëâ If you don't do this, automated bots attack you in minutes.

7Ô∏è‚É£ Processes and systemd (Fundamental)

In Linux, your app is a process.

You need it to:

  • Start on reboot
  • Restart if it fails
  • Have logs
  • Run in production mode

That's where systemd comes in.

Conceptual example:

ExecStart=/usr/bin/node server.js
Restart=always
WorkingDirectory=/var/www/app

üëâ This is real production.

13Ô∏è‚É£ Serverless (Lambdas)

Instead of a server:

  • Upload function
  • AWS executes when called
  • Pay per execution
  • Don't manage operating system

Advantages:

  • Automatic scaling
  • No server maintenance

Limitations:

  • Cold starts
  • Max execution time
  • Less control
